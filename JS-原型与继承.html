JS原型与继承
2023-09-15 23:57:11 #JS #原型 #继承

构造函数与原型

  • 每个函数都有一个 prototype（原型）属性，这个属性是一个指针，指向函数的原型对象，使用原型对象可以让所有对象实例共享它所包含的属性和方法
  • 所有函数的原型对象都会自动获得一个 constructor（构造函数）属性，这个属性指向 prototype 属性所在函数（Person.prototype.constructor -> Person）
  • 构造函数的原型对象只会默认取得 constructor 属性，其他方法都是从 Object 继承而来的
  • 当调用构造函数创建一个新实例后，该实例的内部包含一个指针（内置属性 [[prototype]]），使用 __proto__ 属性可以访问到这个内置属性，__proto__ 指向构造函数的原型对象（p.__proto__ -> Person.prototype）
  • 内置方法 Object.setPrototypeOf 传入两个对象作为参数，将第二个对象设置为第一个对象的原型

实例属性

  • 把函数作为构造函数调用时
  • 在构造函数内部，this 指向新创建的对象，所以在构造函数内部添加的属性直接在新创建的实例上
  • 当通过实例访问构造函数内部的属性时，不需要遍历原型链
  • 每一个实例分别获得在构造函数内创建的属性版本，但是它们都可以访问同一个原型属性

动态特性的副作用

  • 对象与函数原型之间的引用关系是在对象创建时建立的
  • 函数的原型可以被任意替换，已经创建的实例引用旧的原型，新创建的实例引用新的原型

通过构造函数实现对象类型

  • 通过使用 constructor 属性可以访问创建该对象时所用的函数
  • 所有实例对象都可以使用 constructor 属性验证其原始类型
  • 由于 constructor 属性仅仅是原始构造函数的引用，因此可以使用该属性创建新的对象

原型链

  • 每个对象都含有原型的引用，当查找属性时，若对象本身不具有该属性，则会查找原型上是否有该属性
  • 每个对象都可以有一个原型，每个对象的原型也可以有一个原型，以此类推，形成一个原型链
  • 查找特定属性将会被委托在整个原型链上，只有当没有更多的原型可以进行查找时，才会停止查找

实现继承

  • 上述例子中，当通过 dong 对象访问 dance 方法时，JavaScript 运行时将会首先查找 dong 对象本身，由于 dong 对象本身不具有 dance 方法，接下来搜索 dong 对象的原型即 Person 对象，Person 对象也不具有 dance 方法，所以再接着查找 Person 对象的原型，最终找到了 dance 方法

  • Dong 的原型是 Person 对象，故而 Dong 具有 Person 的全部方法

  • 所有继承函数的原型将实时更新，从原型继承的对象总是可以访问当前原型对象

  • 执行 instanceof 操作符，可以判断函数是否继承原型链上的对象功能

注意！上述例子中，通过设置 Person 实例对象作为 Dong 构造函数的原型时，已经丢失了 Dong 和 Dong 初始原型之间的关联

使用 Object.defineProperty 方法修改对象属性的配置信息，因此在 Dong.prototype 对象上增加新的 constructor 属性可以解决上述例子中 constructor 属性丢失的问题
